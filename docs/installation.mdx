## npm

<Tabs>
  <Tab title="npx">
    ```bash
    # PostgreSQL example
    npx badger-db-mcp \
      --transport http \
      --port 8080 \
      --dsn "postgres://user:password@localhost:5432/dbname?sslmode=disable"

    # Or use demo mode for testing
    npx badger-db-mcp --transport http --port 8080 --demo
    ```
  </Tab>

  <Tab title="Global Installation">
    ```bash
    # Install globally
    npm install -g badger-db-mcp

    # Run the server
    badger-db-mcp --transport http --port 8080 --dsn "postgres://..."

    # Or use demo mode for testing
    badger-db-mcp --transport http --port 8080 --demo
    ```
  </Tab>
</Tabs>

## Docker

<Tabs>
  <Tab title="Docker Run">
    ```bash
    docker run --rm --init \
      --name badger-db-mcp \
      --publish 8080:8080 \
      badger-db-mcp \
      --transport http \
      --port 8080 \
      --dsn "postgres://user:password@localhost:5432/dbname?sslmode=disable"

    # Or use demo mode for testing
    docker run --rm --init \
      --name badger-db-mcp \
      --publish 8080:8080 \
      badger-db-mcp \
      --transport http \
      --port 8080 \
      --demo
    ```

    <Warning>
    When connecting to databases on your host machine from Docker, use `host.docker.internal` instead of `localhost`:

    ```bash
    --dsn "postgres://user:password@host.docker.internal:5432/dbname"
    ```
    </Warning>
  </Tab>

  <Tab title="Docker Compose">
    For development environments using Docker Compose, add Badger DB MCP to your `docker-compose.yml`:

    ```yaml docker-compose.yml
    services:
      badger-db-mcp:
        image: badger-db-mcp:latest
        container_name: badger-db-mcp
        ports:
          - "8080:8080"
        environment:
          - BADGER_DB_MCP_LOG_LEVEL=info
        command:
          - --transport
          - http
          - --port
          - "8080"
          - --dsn
          - "postgres://user:password@database:5432/dbname"
        depends_on:
          - database

      database:
        image: postgres:15-alpine
        environment:
          POSTGRES_PASSWORD: password
          POSTGRES_DB: dbname
    ```
  </Tab>
</Tabs>

## Client Integration

Once Badger DB MCP is installed and running, configure your MCP client to connect to it.

### Claude Code

<Tabs>
  <Tab title="CLI">
    <Tabs>
      <Tab title="Stdio">
      For local database connections, use stdio transport:

      ```bash
      claude mcp add --transport stdio badger-db-mcp -- npx badger-db-mcp --transport stdio --dsn "postgres://user:password@localhost:5432/dbname"
      ```

      <Note>
      The `--` separates Claude's flags from Badger DB MCP's arguments. Everything after `--` is the Badger DB MCP command.
      </Note>
      </Tab>
      <Tab title="HTTP">
      For remote or shared database servers, use HTTP transport:

      ```bash
      claude mcp add --transport http badger-db-mcp http://localhost:8080/mcp
      ```

      This connects to a Badger DB MCP server running with HTTP transport:

      ```bash
      npx badger-db-mcp --transport http --port 8080 --dsn "postgres://user:password@localhost:5432/dbname"
      ```
      </Tab>
    </Tabs>
  </Tab>

  <Tab title="Project Configuration">
    For project-specific database connections shared with your team, create `.mcp.json` in your project root.

    <Tabs>
      <Tab title="Stdio">
      ```json .mcp.json
      {
        "mcpServers": {
          "badger-db-mcp": {
            "type": "stdio",
            "command": "npx",
            "args": [
              "badger-db-mcp",
              "--transport",
              "stdio",
              "--dsn",
              "postgres://user:password@localhost:5432/dbname"
            ]
          }
        }
      }
      ```

      <Tip>
      Use environment variable expansion for sensitive values:
      ```json .mcp.json
      {
        "mcpServers": {
          "badger-db-mcp": {
            "type": "stdio",
            "command": "npx",
            "args": [
              "badger-db-mcp",
              "--transport",
              "stdio",
              "--dsn",
              "${DATABASE_URL}"
            ]
          }
        }
      }
      ```
      </Tip>
      </Tab>
      <Tab title="HTTP">
      ```json .mcp.json
      {
        "mcpServers": {
          "badger-db-mcp": {
            "type": "http",
            "url": "http://localhost:8080/mcp"
          }
        }
      }
      ```
      </Tab>
    </Tabs>
  </Tab>

  <Tab title="User Configuration">
    For personal configuration across all projects:

    ```bash
    claude mcp add --scope user badger-db-mcp -- npx badger-db-mcp --transport stdio --dsn "postgres://user:password@localhost:5432/dbname"
    ```

    Or manually edit the user configuration file:

    <Tabs>
      <Tab title="MacOS/Linux">
        Edit `~/.claude/settings.json` and add to the `mcpServers` section.
      </Tab>
      <Tab title="Windows">
        Edit `%USERPROFILE%\.claude\settings.json` and add to the `mcpServers` section.
      </Tab>
    </Tabs>
  </Tab>
</Tabs>

**Using Badger DB MCP:**

Start Claude Code in your project directory:

```bash
claude
```

Verify Badger DB MCP is loaded:

```bash
/mcp
```

### Dify

Badger DB MCP must be running as an HTTP server for Dify to connect. Start Badger DB MCP with HTTP transport:

```bash
npx badger-db-mcp --transport http --port 8080 --dsn "postgres://user:password@localhost:5432/dbname"
```

**Configuration:**

1. In Dify, navigate to **Tools → MCP**
2. Click **Add MCP Server (HTTP)**
3. Configure the server:
   - **Server URL**: `http://localhost:8080/mcp` (or your Badger DB MCP host)
   - **Name**: `Badger DB MCP`
   - **Server Identifier**: `badger-db-mcp` (permanent, max 24 characters)

4. Click save. Dify will automatically discover Badger DB MCP's tools.

**Using Badger DB MCP Tools:**

Once connected, Badger DB MCP tools are available in:

- **Agents**: Tools appear alongside built-in tools, use "Add All" to enable all Badger DB MCP tools
- **Workflows**: Tools appear as available node types

**Multi-Database Setup:**

For multiple databases, use a config file:

```bash
npx badger-db-mcp --transport http --port 8080 --config /path/to/dbhub.toml
```

**Demo Mode:**

For testing, use the built-in demo database:

```bash
npx badger-db-mcp --transport http --port 8080 --demo
```

**Limitations:**
- Dify only supports HTTP transport (not stdio)
- Badger DB MCP must be network-accessible from Dify

**References:**
- [Dify MCP Documentation](https://docs.dify.ai/en/guides/tools/mcp)

### LibreChat

Badger DB MCP can be integrated with [LibreChat](https://www.librechat.ai/) using either **stdio** or **Streamable HTTP** transport.

<Tabs>
  <Tab title="stdio">
    ```yaml librechat.yaml
    mcpServers:
      badger-db-mcp:
        type: stdio
        command: npx
        args:
          - -y
          - "badger-db-mcp"
          - "--dsn"
          - "postgres://user:password@localhost:5432/dbname"
        timeout: 60000
    ```

    For demo mode:

    ```yaml librechat.yaml
    mcpServers:
      badger-db-mcp:
        type: stdio
        command: npx
        args:
          - -y
          - "badger-db-mcp"
          - "--demo"
        timeout: 60000
    ```
  </Tab>

  <Tab title="Streamable HTTP">
    ```yaml librechat.yaml
    mcpServers:
      badger-db-mcp:
        type: streamable-http
        url: http://host.docker.internal:8080/mcp
        timeout: 60000
    ```

    <Note>
      **Docker Networking:** When LibreChat runs in Docker (e.g., with docker-compose), use `host.docker.internal:8080` to connect to Badger DB MCP running on your host machine. This is Docker's special DNS name that resolves to the host's IP address.

      For native installations, use `http://localhost:8080/mcp` instead.
    </Note>
  </Tab>
</Tabs>

**References:**
- [LibreChat MCP Documentation](https://www.librechat.ai/docs/configuration/librechat_yaml/object_structure/mcp_servers)
